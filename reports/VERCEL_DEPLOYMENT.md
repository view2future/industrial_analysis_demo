# 部署到Vercel

注意：Vercel主要针对前端应用，对Python后端支持有限。本项目是一个完整的Flask Web应用，更适合部署到Render或Heroku等平台。

## 限制说明

1. **冷启动**: Vercel的无服务器函数有冷启动延迟
2. **执行时间**: 每个函数有最大执行时间限制（通常10秒）
3. **文件系统**: 无服务器环境没有持久化文件系统
4. **并发**: 无服务器函数的并发处理能力有限

## 部署步骤

### 1. 重构应用
需要将Flask应用重构为Vercel无服务器函数：
1. 将每个路由转换为独立的函数
2. 使用Vercel的文件系统API替代本地文件操作
3. 使用外部数据库替代SQLite

### 2. 创建无服务器函数
在`api/`目录下创建函数文件：
```
api/
├── index.py          # 主页和路由处理
├── upload.py         # 文件上传处理
├── analysis.py       # 分析功能
└── ...
```

### 3. 配置Vercel
1. 在Vercel仪表板中导入GitHub仓库
2. 设置环境变量
3. 配置构建和输出设置

## 推荐替代方案

由于本项目的复杂性，强烈推荐使用以下平台：
1. **Render** - 专门支持Python Web应用
2. **Heroku** - 成熟的云平台即服务
3. **Railway** - 现代化的部署平台
4. **AWS Elastic Beanstalk** - AWS的平台即服务
5. **Google Cloud Run** - Google的容器化部署服务

这些平台更适合部署完整的Flask应用，提供更好的性能和更简单的配置。
